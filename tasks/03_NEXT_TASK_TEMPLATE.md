# Next Task Template

아래 템플릿만 채우면 다음 작업을 바로 시작할 수 있습니다.

---

## 1) 목표

- 무엇을 개선/수정할지 1~2문장

## 2) 사용자 체감 문제

- 현재 불편/오류
- 재현 경로

## 3) 범위

- IN SCOPE:
- OUT OF SCOPE:

## 4) 수정 대상 파일(예상)

- `src/...`
- `src/...`

## 5) 구현 체크리스트

- [ ] 코드 경로 탐색 및 원인 확인
- [ ] UI/로직 수정
- [ ] 관련 테스트 또는 최소 빌드 검증
- [ ] 회귀 영향 확인
- [ ] 커밋/푸시

## 6) 검증 커맨드

```bash
npm run build
npm run test -- <관련 테스트 파일>
```

## 7) 커밋 메시지(안)

- `fix(...): ...`
- `feat(...): ...`
- `style(...): ...`

## 8) 완료 기준(Definition of Done)

- [ ] 사용자가 제시한 재현 케이스에서 문제 미발생
- [ ] 빌드 통과
- [ ] 원격 브랜치 동기화 확인 (`git rev-list --left-right --count HEAD...origin/main` = `0 0`)
